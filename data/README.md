# data/ — Price Data and Scenario Generation

This module generates stochastic price scenarios for the energy and regulation markets, providing the uncertain inputs that drive the EMS optimizer.

---

## Price Model

### Base Energy Price Profile

The deterministic base profile combines:

```
price_energy(t) = base + daily_cycle + evening_peak + morning_peak
```

Where:
```
base           = 0.050  [$/kWh]                                   Constant floor
daily_cycle    = 0.025 · sin(2π(t − 6) / 24)                      Sinusoidal day/night variation
evening_peak   = 0.040 · exp(−0.5 · ((t mod 24 − 18) / 2)²)      Gaussian peak at 6 PM
morning_peak   = 0.015 · exp(−0.5 · ((t mod 24 − 8) / 1.5)²)     Gaussian peak at 8 AM
```

The profile is floored at $0.005/kWh (prices cannot go below near-zero). This creates a realistic daily pattern with:
- Low prices overnight (~$0.03/kWh)
- Morning demand peak (~$0.07/kWh)
- Midday moderate prices (~$0.05/kWh)
- Evening demand peak (~$0.09/kWh) — the highest of the day

### Regulation Price Profile

Regulation capacity prices are correlated with energy prices but typically lower:

```
price_reg(t) = 0.4 · price_energy(t) + 0.01 + noise
noise ~ N(0, 0.003²)
```

Floored at $0.002/kW/h. The 0.4x correlation reflects the empirical observation that regulation prices tend to be higher when energy prices are high (grid stress periods).

## Stochastic Scenarios

The EMS uses 5 price scenarios to hedge against forecast uncertainty:

| Scenario | Probability | Perturbation | Interpretation |
|----------|-------------|-------------|----------------|
| 0 (base) | 0.40 | 0% | Most likely: prices follow forecast |
| 1 | 0.20 | +15% | Moderate upside: higher than expected |
| 2 | 0.20 | −15% | Moderate downside: lower than expected |
| 3 | 0.10 | +30% | High upside: price spike |
| 4 | 0.10 | −30% | High downside: price collapse |

### Scenario Construction

Each scenario applies a multiplicative perturbation to the base profile:

```
price_scenario[s, t] = price_base[t] · (1 + scale[s] · noise_smoothed[t])
```

Where `noise_smoothed` is **time-correlated** Gaussian noise, generated by applying an exponential moving average (EMA) to white noise:

```
raw ~ N(0, 1)
smoothed[0] = raw[0]
smoothed[t] = α · raw[t] + (1 − α) · smoothed[t−1],    α = 1/(τ + 1),  τ = 3 h
```

The smoothed noise is then normalized to unit variance. The 3-hour correlation time means that price deviations persist for several hours rather than jumping randomly each hour — matching real market behavior.

Energy and regulation scenarios use **independent** noise realizations but the same perturbation structure, reflecting that while both markets are correlated in level, their random fluctuations are partially independent.

## Files

## Interpolation

`PriceGenerator.interpolate_to_mpc()` provides zero-order hold interpolation from hourly prices to MPC resolution (60 s):

```
price_mpc[k] = price_hourly[⌊k / 60⌋]
```

Each hourly price is repeated 60 times. This is used when MPC-resolution price data is needed for real-time profit calculation.
